具体例子：一家 50 人公司

场景

公司有 50 个员工，每人每天下载 5 张内部图片（产品图、设计稿等），系统自动嵌入水印追踪。

计算

  • 每月调用：50 人 x 5 张 x 22 天 = 5,500 次
  • Lambda 计算：5,500 x 6 GB-s = 33,000 GB-s < 400,000 免费额度
  • S3 存储：假设平均 2MB/图，共 100 张原图 = 200MB
  • S3 请求：5,500 GET + 5,500 PUT = 11,000 次

月费用

  • Lambda：$0（在免费额度内）
  • S3 存储：200MB x $0.025/GB = $0.005
  • S3 请求：11,000 x $0.0000004 = $0.004
  • 总计：不到 $0.01/月

另一个例子：图片版权保护平台

场景

一个摄影素材网站，500 个付费用户，每人每天下载 10 张高清图。网站需要追踪谁泄露了图片。

计算

  • 每月调用：500 人 x 10 张 x 30 天 = 150,000 次
  • Lambda 计算：150,000 x 6 GB-s = 900,000 GB-s
  • 超出免费额度：900,000 - 400,000 = 500,000 GB-s
  • S3 存储：10,000 张原图 x 5MB = 50GB

月费用

  • Lambda 请求：50,000 x $0.20/百万 = $0.01
  • Lambda 计算：500,000 GB-s x $0.0000166667 = $8.33
  • S3 存储：50GB x $0.025 = $1.25
  • S3 请求：300,000 x $0.0000004 = $0.12
  • 总计：约 $9.71/月

对比商业水印 SaaS 每月几百到几千美金的订阅费，自建方案便宜得多。

优化方式：

  • • 定时 ping 保持 Lambda 热（每 5 分钟一次）
  • • 增加 Lambda 内存（2048MB → 3008MB，获得更多 CPU）
  • • Provisioned Concurrency（消除冷启动，~$15/月）
  • • 使用较小的图片尺寸

为什么这么便宜？

1

Serverless 架构

不用的时候不花钱。没有请求 = 没有费用。不需要 24/7 运行的服务器。

2

Lambda 免费额度大

每月 100 万次请求 + 400K GB-s 免费。绝大多数用例完全不超。

3

共享基础设施

EC2、ALB、CloudFront 是已有的共享资源，不产生增量成本。

4

纯算法，无外部依赖

水印算法只用 numpy + opencv，不调用任何付费 AI/ML 服务。
