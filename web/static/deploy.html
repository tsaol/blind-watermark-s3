部署指南

从零开始部署盲水印方案到你的 AWS 账号

前置条件

  • ✓ AWS 账号，具备 CloudFormation / Lambda / S3 / IAM 权限
  • ✓ AWS CLI 已安装并配置（aws configure）
  • ✓ Python 3.12+
  • ✓ 一台 EC2 实例（用于 Web 服务，可选）

Step 1: 克隆仓库

获取源代码

git clone https://github.com/tsaol/blind-watermark-s3.git

cd blind-watermark-s3

Step 2: 构建 Lambda Layer

打包 numpy + opencv 依赖

# 创建构建目录

mkdir -p build/layer/python

# 安装依赖到 layer 目录

pip install numpy opencv-python-headless Pillow \

  -t build/layer/python \

  --platform manylinux2014_x86_64 \

  --only-binary=:all:

# 打包为 zip

cd build/layer && zip -r9 ../../layer.zip python/

cd ../..

Step 3: 上传部署产物到 S3

Lambda 代码和 Layer 需要先上传到 S3

# 设定变量

REGION=ap-northeast-1

BUCKET=your-artifact-bucket # 替换为你的桶名

# 创建 S3 桶（如果不存在）

aws s3 mb s3://$BUCKET --region $REGION

# 打包 Lambda 代码

cd src/embed && zip -r ../../embed.zip . && cd ../..

cd src/extract && zip -r ../../extract.zip . && cd ../..

# 复制 shared watermark.py 到两个函数

cp src/shared/watermark.py src/embed/

cp src/shared/watermark.py src/extract/

# 上传到 S3

aws s3 cp layer.zip s3://$BUCKET/deploy/layer.zip --region $REGION

aws s3 cp embed.zip s3://$BUCKET/deploy/embed.zip --region $REGION

aws s3 cp extract.zip s3://$BUCKET/deploy/extract.zip --region $REGION

Step 4: 部署 CloudFormation Stack

一键创建所有 AWS 资源

aws cloudformation deploy \

  --template-file template.yaml \

  --stack-name blind-watermark \

  --capabilities CAPABILITY_NAMED_IAM \

  --region $REGION \

  --parameter-overrides \

    ArtifactBucket=$BUCKET \

    ArtifactPrefix=deploy

这会创建：

  • • S3 桶（存储原始图片）
  • • Lambda 函数 x 2（嵌入 + 提取）
  • • Lambda Function URLs（IAM 认证）
  • • IAM Role + Policy
  • • Lambda Layer（numpy + opencv）

Step 5: 验证部署

确认 Lambda 函数已正确创建

# 列出函数

aws lambda list-functions --region $REGION \

  --query 'Functions[?starts_with(FunctionName,`blind-watermark`)].FunctionName'

# 应输出:

# ["blind-watermark-embed", "blind-watermark-extract"]

Step 6: 测试水印功能

用 CLI 直接测试嵌入和提取

# 上传一张测试图片到 S3

aws s3 cp test.jpg s3://YOUR-SOURCE-BUCKET/tmp/test.jpg \

  --region $REGION

# 调用嵌入 Lambda

aws lambda invoke --function-name blind-watermark-embed \

  --region $REGION \

  --payload '{"body":"{\"key\":\"tmp/test.jpg\",\"user_id\":\"alice\"}"}' \

  output.json

# output.json 包含 base64 编码的水印图片

# 解码并保存

python3 -c " import json, base64 data = json.load(open('output.json')) img = base64.b64decode(data['body']) open('watermarked.png','wb').write(img) print('Saved watermarked.png') "

# 调用提取 Lambda

python3 -c " import json, base64, subprocess img_b64 = base64.b64encode(open('watermarked.png','rb').read()).decode() payload = json.dumps({'body': json.dumps({'image': img_b64})}) open('/tmp/extract_payload.json','w').write(payload) "

aws lambda invoke --function-name blind-watermark-extract \

  --region $REGION \

  --payload file:///tmp/extract_payload.json \

  extract_result.json

cat extract_result.json

# 应输出: {"user_id": "alice", "confidence": ...}

Step 7: 部署 Web UI（可选）

在 EC2 上运行可视化演示页面

# SSH 到你的 EC2 实例

ssh ubuntu@your-instance

# 克隆代码

git clone https://github.com/tsaol/blind-watermark-s3.git

cd blind-watermark-s3/web

# 创建虚拟环境并安装依赖

python3 -m venv venv

source venv/bin/activate

pip install -r requirements.txt

# 设置环境变量

export SOURCE_BUCKET=your-source-bucket-name

export AWS_DEFAULT_REGION=ap-northeast-1

# 用 gunicorn 启动

gunicorn -w 2 -b 0.0.0.0:8090 server:app

Systemd 服务（生产环境推荐）

[Unit]

Description=Blind Watermark Web

After=network.target

[Service]

User=ubuntu

WorkingDirectory=/home/ubuntu/blind-watermark

ExecStart=/home/ubuntu/blind-watermark/venv/bin/gunicorn \

  -w 2 -b 0.0.0.0:8090 server:app

Restart=always

[Install]

WantedBy=multi-user.target

部署架构总览

│  浏览器   │────▶│  Web 服务  │────▶│  Lambda (嵌入)   │────▶│ S3  │
│          │     │  Flask    │     │  DCT 频域编码    │     │原始图│
└──────────┘     │  :8090    │     └─────────────────┘     └─────┘
                 │           │
                 │           │────▶┌─────────────────┐
                 └───────────┘     │  Lambda (提取)   │
                                   │  多数投票解码    │
                                   └─────────────────┘

• Lambda Function URLs 使用 IAM 认证，无公开端点

• Web 服务通过 AWS SDK 调用 Lambda，需要 IAM 凭证

• 所有流量通过 HTTPS

常见问题

Q: Lambda 超时？

A: 确保内存至少 2048MB，超时设为 120s。大图处理需要更多内存。

Q: Layer 太大？

A: 用 opencv-python-headless 而不是 opencv-python，可以省 ~100MB。

Q: 提取出来的 user_id 是乱码？

A: 图片太小（需要至少 256x256）或经过了严重压缩（JPEG quality < 50）。

Q: Web UI 无法调用 Lambda？

A: EC2 实例需要附加 IAM Role，允许 lambda:InvokeFunction 和 s3:PutObject/GetObject。
